# 实验方案总体设计

**核心目标**：验证所提出的 **TruthRide** 在群智感知任务中，能否在存在虚假数据和工人流失的情况下，实现：

1. 更高的数据质量；
2. 更稳定的任务完成能力；
3. 更低的可信工人流失；
4. 更优的质量-成本收益。

---

# 一、对比算法 Baselines

| 策略                      | 含义                         | 对比目的                           |
| ----------------------- | -------------------------- | ------------------------------ |
| Random                  | 随机选择工人                     | 下限基准                           |
| Explore-First / ε-First | 前期随机探索，后期利用历史均值            | 对比简单探索-利用策略                    |
| UCB-Greedy / CMAB       | 基于 UCB 和边际收益/成本的工人招募       | 验证基础 CMAB 招募效果                 |
| Trust-Aware             | CMAB + 空间一致性信任验证           | 验证 trust verification 对数据质量的贡献 |
| Membership-Aware        | CMAB + Trust + PGRD        | 验证会员机制对平台收益和参与积极性的贡献           |
| TruthRide               | CMAB + Trust + PGRD + LGSC | 完整方案，验证长期留存和综合效果               |

---

# 二、评价指标 Metrics

## 1. 数据质量指标

| 指标                         | 含义              | 作用                         |
| -------------------------- | --------------- | -------------------------- |
| Per-round Average Quality  | 每轮收到数据的平均质量     | 展示动态质量变化                   |
| Cumulative Average Quality | 累计平均数据质量        | 主指标，展示长期数据质量               |
| RMSE / Error               | 上报数据与可信基准值之间的误差 | 如果你后面能从 `task_data` 算，建议加入 |

---

## 2. 任务执行指标

| 指标                          | 含义                 | 作用            |
| --------------------------- | ------------------ | ------------- |
| Cumulative Completion Ratio | 同时满足人数要求和质量阈值的任务比例 | 证明质量提升没有牺牲完成率 |
| Cumulative Coverage Ratio   | 被至少一个工人覆盖的任务比例     | 证明没有只挑少量高质量任务 |

---

## 3. 平台经济指标

| 指标                      | 公式                                                           | 作用            |
| ----------------------- | ------------------------------------------------------------ | ------------- |
| Data Value              | ρ × Σ w_j q_j                                                | 衡量平台收到数据本身的价值 |
| Platform Utility        | Data Value + Membership Fee − Worker Payment − Bonus Payment | 衡量扣除成本后的净效益   |
| Cost-Quality Efficiency | Cumulative Weighted Quality / Cumulative Payment             | 衡量单位成本换来的数据质量 |

你现在的累计平台效益可以保留，但论文里要明确它是**净效益**，不是单纯数据价值。

---

## 4. 留存与激励指标

| 指标                                 | 含义          | 作用             |
| ---------------------------------- | ----------- | -------------- |
| Cumulative Left Workers            | 累计流失工人数     | 验证 LGSC 是否降低流失 |
| High-quality Worker Retention Rate | 高质量/可信工人留存率 | 更精准证明留住了可信工人   |
| Member Count                       | 会员工人数       | 验证 PGRD 入会效果   |
| Trusted Member Count               | 可信会员人数      | 验证会员机制是否吸引可信工人 |

---

## 5. 信任识别指标

| 指标                                 | 含义              | 作用            |
| ---------------------------------- | --------------- | ------------- |
| Average Trust of Trusted Workers   | 真实可信工人的平均 trust | 应该上升或保持高位     |
| Average Trust of Malicious Workers | 真实恶意工人的平均 trust | 应该下降          |
| Malicious Detection F1-score       | 系统识别恶意工人的准确性    | 证明 IEWV 问题被解决 |

# 三、实验一：整体性能对比实验

## 目的

验证 TruthRide 在数据质量、完成率、平台效益和工人留存方面的综合优势。

## 数据集

北京出租车、罗马出租车、芝加哥自行车均可使用。建议先用北京作为主实验。

## 对比算法

Random、Explore-First、UCB-Greedy、Trust-Aware、Membership-Aware、TruthRide。

# 六、实验四：参数敏感性实验

## 目的

证明 TruthRide 对关键参数稳定，并找出合理参数区间。

建议先做 3 个参数，不要太多。

| 参数               | 横轴取值               | 观察指标                             | 目的         |
| ---------------- | ------------------ | -------------------------------- | ---------- |
| Membership Fee   | 1, 2, 3, 5, 7      | Utility / Member Count / Quality | 找最优会费区间    |
| Validation Top-M | 0, 5, 10, 15, 20   | Quality / Trust F1-score         | 验证验证任务数量影响 |
| Sunk Threshold   | 20, 30, 40, 50, 60 | Retention / Utility              | 验证沉没成本阈值影响 |
