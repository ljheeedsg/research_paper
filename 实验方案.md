# 移动车辆感知实验方案

> 基于SUMO仿真的轨迹时空段数据采集与算法验证

---

## 一、实验目标

将论文《确定性轨迹实验重现.docx》中的实验设计，通过SUMO交通仿真平台实现：

- 生成车辆轨迹数据（150辆/小时 × 6小时 = 900条轨迹）
- 提取轨迹时空段（6个区域，时间窗口6小时）
- 验证双阶段工人招募算法（CMAB + 空间一致性验证）
- 测试会员激励机制（PGRD + LGSC）

---

## 二、实验参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 区域数目 K | 6 | 感知区域数量 |
| 仿真时长 N | 6小时（21600秒） | 最长感知时间 |
| 车辆流量 | 150辆/小时 | 招募约束 |
| 任务总数 | 120个 | 每小时20个 |
| 车辆总数 | 900辆 | 150×6 = 900 |
| 真/假工人比例 | 70% / 30% | 用于验证识别准确率 |

## 四、第一阶段：模拟数据实验（详细步骤）

### 步骤1：生成模拟车辆轨迹

**目标**：生成900辆车的模拟轨迹

**生成规则**：
```
- 每辆车随机经过 2~4 个区域
- 进入区域时间在整点后 0~600秒 内随机
- 在每个区域停留 60~300秒
- 车辆报价在 [5, 20] 区间随机
- 车辆可信度：70%为真、30%为假
```

**输出格式**（CSV）：
```csv
vehicle_id,regions_visited,total_cost,is_trusted
v001,"[0,2,4]",12.5,true
v002,"[1,3,5]",8.3,false
```

**验收标准**：
- [ ] 成功生成900条车辆记录
- [ ] 每个区域被访问次数在合理范围（100~250次）

---

### 步骤2：生成模拟任务集合

**目标**：生成120个任务，分布在6个区域

**生成规则**：
```
- 每个区域分配 20 个任务
- 任务时间窗口 = 3600秒（1小时）
- 任务随机分配到6小时中的任意时段
- 每个任务需要 1~3 个工人完成
```

**输出格式**（CSV）：
```csv
task_id,region_id,start_time,end_time,required_workers
t001,0,0,3600,2
t002,1,300,2700,1
```

**验收标准**：
- [ ] 成功生成120条任务记录
- [ ] 6个区域任务数均衡（各20个左右）

### 步骤6：实现CMAB招募算法（第一阶段）

**目标**：根据论文公式12，实现贪心招募

**算法流程**：
```
输入：任务时空段、工人轨迹时空段、预算

1. 计算每个工人能覆盖哪些任务
2. 按「覆盖任务数/报价」计算性价比
3. 贪心选择：反复选性价比最高的工人，直到预算耗尽
4. 输出：被选中的工人集合
```

**验收标准**：
- [ ] 招募结果不为空
- [ ] 总成本 ≤ 预算
- [ ] 任务覆盖数 > 0

---

### 步骤7：实现空间一致性验证（第二阶段）

**目标**：验证工人数据的可信度

**算法流程**：
```
输入：可信工人集合 Uc、未验证工人集合 Uu

1. 将地图划分为 N×N 的grid
2. 计算 Uc 和 Uu 的共同访问区域
3. 选取 top-M 个高密度grid作为验证区
4. 模拟验证：对比 Uu 工人在验证区的数据与 Uc 的重合度
5. 更新工人信任度
```

**验收标准**：
- [ ] 验证后 Uc 集合增大
- [ ] 假工人被正确识别

---

### 步骤8：实现会员激励机制

**目标**：测试会员机制对工人留存的影响

**算法内容**：
```
PGRD（参照依赖效用）：
- 会员报酬 = 基础报酬 × 会员等级系数
- 非会员报酬 = 基础报酬

LGSC（沉没成本激励）：
- 历史贡献越多，退出惩罚越高
- 长期可信工人获得额外奖励
```

**验收标准**：
- [ ] 会员工人留存率 > 非会员
- [ ] 激励机制不影响任务完成率

---

### 步骤9：计算评估指标

**目标**：输出5个核心指标

| 指标 | 计算方式 | 目标值 |
|------|----------|--------|
| 任务完成率 | 被覆盖任务数 / 总任务数 | > 80% |
| 数据质量 | 正确识别的可信工人数 / 总工人数 | > 85% |
| 总成本 | 所有招募成本之和 | 尽量低 |
| 可信工人留存率 | 验证后仍在 Uc 的工人数 / 初始 Uc 数 | > 70% |
| 运行时间 | 算法总执行时间 | < 10秒 |

---

### 步骤10：对比基线算法

**目标**：验证本论文方法优于基线

| 方法 | 实现方式 |
|------|----------|
| Random | 随机选取工人 |
| Greedy-Cost | 只选报价最低的 |
| CMAB-Baseline | 只用第一阶段招募 |
| 本论文方法 | 双阶段 + 会员激励 |

**验收标准**：
- [ ] 4种方法都能正常运行
- [ ] 本论文方法在至少3个指标上优于基线
