# 数据预处理脚本 – 代码文档

## 概述

本脚本将原始出租车温度数据集 `crowd_temperature.csv` 处理为算法可直接使用的车辆轨迹段文件（`step6_vehicle.csv`），并生成网格分布可视化图（`step6_grid_partition.png`）。处理流程包括：日期筛选、时间转换、网格划分、轨迹段构建、跨小时拆分、车辆ID重置、添加成本和可信属性。

**用途**：为后续任务生成、工人招募算法提供基础时空轨迹数据。

---

## 输入文件

| 文件名 | 格式 | 说明 |
|--------|------|------|
| `dataset/crowd_temperature.csv` | CSV | 原始出租车轨迹数据，包含字段：出租车ID、日期、时间、纬度、经度、温度。 |

### 输入文件结构示例

```
taxi_id,date,time,lat,lon,temp
1234,02-01-14,08:15:23.456+01,48.1234,11.5678,22.5
...
```

- `taxi_id`：出租车标识符
- `date`：日期，格式 `日-月-年`
- `time`：时间，含时区和毫秒，如 `08:15:23.456+01`
- `lat`：纬度
- `lon`：经度
- `temp`：温度（本脚本未使用温度值，仅保留用于未来扩展）

---

## 输出文件

| 文件名 | 格式 | 说明 |
|--------|------|------|
| `step6_vehicle.csv` | CSV | 车辆轨迹段文件，每条记录为一个不跨小时的连续轨迹段。字段：`vehicle_id, region_id, start_time, end_time, cost, is_trusted` |
| `step6_grid_partition.png` | PNG | 两张子图：左侧为原始GPS点散点图+网格线，右侧为每个网格内的点密度热力图。 |

### `step6_vehicle.csv` 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `vehicle_id` | string | 格式 `v{小时:02d}_{车辆序号:03d}`，例如 `v08_042`，其中小时为轨迹段起始时间所在小时 |
| `region_id` | int | 网格编号 0~99（10×10 网格，按行主序排列） |
| `start_time` | int | 轨迹段起始时间，从午夜开始的秒数（0~86400） |
| `end_time` | int | 轨迹段结束时间（秒），`end_time > start_time`，且不跨小时边界 |
| `cost` | float | 车辆单位成本，范围 `[COST_MIN, COST_MAX]`，随机生成 |
| `is_trusted` | bool | 是否为可信车辆，按 `TRUSTED_RATIO` 随机生成 |

## 配置参数（脚本顶部可修改）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `INPUT_FILE` | `'dataset/crowd_temperature.csv'` | 原始数据文件路径 |
| `OUTPUT_CSV` | `'step6_vehicle.csv'` | 输出轨迹段CSV文件名 |
| `OUTPUT_PLOT` | `'step6_grid_partition.png'` | 输出图片文件名 |
| `TARGET_DATE` | `'02-01-14'` | 筛选的目标日期（日-月-年） |
| `GRID_X_NUM` | `10` | 经度方向网格数量 |
| `GRID_Y_NUM` | `10` | 纬度方向网格数量 |
| `COST_MIN` | `5.0` | 车辆成本最小值 |
| `COST_MAX` | `20.0` | 车辆成本最大值 |
| `TRUSTED_RATIO` | `0.5` | 可信车辆比例（0~1） |
| `random.seed(3)` | 固定种子 | 保证成本和可信属性的随机结果可重复 |

## 处理流程详解

### 步骤1：读取、筛选、时间转换

- 读取 `crowd_temperature.csv`，跳过表头。
- 筛选出日期等于 `TARGET_DATE` 的行。
- 将时间字符串（如 `08:15:23.456+01`）转换为从午夜开始的秒数：
  - 去掉时区（`+01`）和毫秒（`.456`）
  - 解析 `时:分:秒` 并计算 `秒 = 时*3600 + 分*60 + 秒`
- 提取经纬度，用于后续范围计算。
- 存储每一条记录：`(taxi_id, time_sec, lat, lon, temp)`。

### 步骤2：计算经纬度范围并划分网格

- 计算所有点的 `lon_min, lon_max, lat_min, lat_max`。
- 根据 `GRID_X_NUM` 和 `GRID_Y_NUM` 计算每个网格的尺寸：
  ```
  grid_size_lon = (lon_max - lon_min) / GRID_X_NUM
  grid_size_lat = (lat_max - lat_min) / GRID_Y_NUM
  ```
- 为每个点分配 `region_id`：
  ```
  ix = floor((lon - lon_min) / grid_size_lon)
  iy = floor((lat - lat_min) / grid_size_lat)
  region_id = iy * GRID_X_NUM + ix
  ```
  - 边界处理：若 `ix == GRID_X_NUM` 则减1，`iy` 同理。
- 统计每个网格的点数 `grid_counts`。

### 步骤3：构建基本轨迹段（连续不重叠）

- 按 `taxi_id` 分组，每组内按时间排序。
- 对于每辆车，遍历连续的时间点：
  - 第一段：起始时间为第一个点的时间。
  - 后续段：起始时间 = 上一段的结束时间 + 1 秒（避免重叠）。
  - 结束时间 = 下一个点的时间。
  - 轨迹段所属区域 = 当前点（起点）的 `region_id`。
- 得到基本段列表：`(taxi_id, region, start, end)`。

### 步骤4：按小时切分（跨整点拆分）

- 生成整点边界列表：`3600, 7200, ..., 82800`。
- 对每个基本段，若 `start // 3600 != end // 3600`（即跨小时），则拆分为多个子段，每个子段完全落在一个小时内。
- 拆分函数 `split_across_hours(s, e)`：
  - 从 `cur_start = s` 开始，找下一个整点边界 `next_boundary = ((cur_start // 3600) + 1) * 3600`。
  - 取 `cur_end = min(e, next_boundary)`，产生子段 `(cur_start, cur_end)`。
  - 更新 `cur_start = cur_end`，重复直到 `cur_start >= e`。
- 若不跨小时，直接保留原段。

### 步骤5：重置车辆ID

- 收集所有唯一的原始 `taxi_id`，按整数排序。
- 建立映射：`new_id = 索引`（从0开始）。
- 将所有轨迹段中的 `taxi_id` 替换为 `new_id`。

### 步骤6：生成成本和可信属性

- 为每个 `new_id`（即每辆车）生成：
  - `cost`：在 `[COST_MIN, COST_MAX]` 内均匀随机，保留1位小数。
  - `is_trusted`：按 `TRUSTED_RATIO` 概率为 `True`，否则 `False`。
- 这些属性在同一辆车的所有轨迹段中保持不变。

### 步骤7：写入最终CSV

- 遍历所有轨迹段 `(new_id, region, start, end)`。
- 计算小时：`hour = start // 3600`。
- 生成 `vehicle_id`：`f"v{hour:02d}_{new_id:03d}"`。
- 写入一行：`vehicle_id, region_id, start_time, end_time, cost, is_trusted`。

### 步骤8：生成网格分布图

- **左子图（散点图）**：
  - 绘制所有原始GPS点（蓝色，小点）。
  - 叠加网格线（根据 `lon_min, lon_max, lat_min, lat_max` 和网格数量）。
  - 坐标轴标签为实际经纬度值。
- **右子图（热力图）**：
  - 创建 `GRID_Y_NUM × GRID_X_NUM` 矩阵，填充每个网格内的点数。
  - 使用 `imshow` 绘制，颜色映射 `hot`。
  - 添加颜色条。
- 保存图片为 `step6_grid_partition.png`，显示或保存。

## 关键函数说明

### `time_to_seconds(t_str)`

- **输入**：时间字符串，如 `"08:15:23.456+01"`
- **处理**：去除时区（`+01`）和毫秒（`.456`），解析 `时:分:秒`。
- **返回**：从午夜开始的秒数（整数）。

### `split_across_hours(s, e)`

- **输入**：起始秒数 `s`，结束秒数 `e`（`s < e`）。
- **输出**：子区间列表 `[(start1, end1), (start2, end2), ...]`，每个子区间不跨小时边界。
- **示例**：`s=3500, e=3700` → `[(3500,3600), (3600,3700)]`。

## 注意事项

1. **日期格式**：脚本假设日期为 `日-月-年` 格式（如 `02-01-14` 表示 2014年1月2日）。若原始数据使用其他格式，需修改比较逻辑。
2. **时间格式**：支持时区和毫秒，脚本会主动去除。若时区符号不是 `+`，可调整 `split('+')[0]` 部分。
3. **轨迹段连续性**：相邻点之间的时间间隔被忽略，仅按顺序连接。若两点时间间隔过大，仍生成一段轨迹（但可能不合理）。可根据需要增加间隔阈值判断。
4. **网格边界处理**：当点恰好落在右边界或上边界时，索引会减1，确保 `region_id` 在 0~99 范围内。
5. **车辆ID重置**：原始 `taxi_id` 可能不是连续整数，重置为 `0,1,2,...` 便于后续处理。
6. **成本和可信属性**：同一辆车所有轨迹段共享相同属性，这是合理的，因为成本与车辆本身相关。
7. **随机种子**：固定为 `3`，保证每次运行生成的 `cost` 和 `is_trusted` 一致。修改种子可改变随机结果。

## 输出数据特点

- **每个轨迹段不跨小时**：方便后续按小时统计工人时空可用性。
- **`vehicle_id` 包含起始小时**：便于识别车辆在哪个小时段活动。
- **`region_id` 为网格编号**：空间离散化为 10×10 网格，与任务生成脚本保持一致。

## 扩展建议

- **支持多日数据**：当前仅处理单日，可修改为按日期循环输出多个文件。
- **添加间隔阈值**：若两个GPS点时间间隔超过某值（如1小时），则断开轨迹段，避免不合理的长段。
- **保留温度信息**：可将温度加入轨迹段，用于模拟感知质量。
- **动态网格划分**：可根据数据分布自适应调整网格边界。
