# **5.X.1 平台收益模型定义**

为更加真实地刻画数据质量对平台收益的影响，本文不再采用传统的线性收益模型（即收益与数据质量成正比），而是引入一种**基于质量阈值的非线性收益函数**。

---

## （1）任务质量定义

对于任务 ( j ) 在第 ( t ) 轮，其数据质量定义为：

[
q_j(t) \in [0,1]
]

其中，( q_j(t) ) 表示该任务所有参与工人上报数据的平均质量（或与可信参考值的一致性）。

---

## （2）任务收益函数

定义任务收益函数为：

[
R(q_j) = R_{max} \cdot \left( \frac{2}{1 + e^{-k(q_j - q_0)}} - 1 \right)
]

---

## （3）参数说明

| 参数          | 含义          |
| ----------- | ----------- |
| ( q_j )     | 任务数据质量      |
| ( q_0 )     | 盈亏平衡点（质量阈值） |
| ( k )       | 曲线陡峭程度      |
| ( R_{max} ) | 最大收益        |

---

## （4）函数特性分析

该函数具有如下性质：

1. **低质量负收益**
   当 ( q_j < q_0 ) 时，( R(q_j) < 0 )，表示低质量数据会误导平台决策，从而带来损失。

2. **收益转折点**
   当 ( q_j = q_0 ) 时，( R(q_j) \approx 0 )，表示数据质量达到最低可接受标准。

3. **快速增长区间**
   当 ( q_j > q_0 ) 时，收益随质量快速增加。

4. **边际收益递减**
   当 ( q_j \to 1 ) 时，收益趋近于 ( R_{max} )，反映高质量数据的边际收益递减。

---

## （5）设计动机

该收益函数能够更加真实地反映如下事实：

* 低质量数据不仅无价值，还可能造成错误决策；
* 高质量数据对平台具有显著价值；
* 数据质量提升存在边际收益递减现象。

---

# **5.X.2 平台净效用定义**

在第 ( t ) 轮，平台净效用定义为：

[
U_t = \sum_{j \in \Gamma_t} R(q_j(t)) + F_t - C_t - B_t
]

---

## 各项说明

| 项               | 含义          |
| --------------- | ----------- |
| ( \sum R(q_j) ) | 所有任务的数据质量收益 |
| ( F_t )         | 本轮会员费收入     |
| ( C_t )         | 工人报酬支出      |
| ( B_t )         | 会员奖励支出      |

---

## 累计平台效用

[
U_{cum}(T) = \sum_{t=1}^{T} U_t
]

## 预期趋势

[
\text{TruthRide} > \text{Incentive-Aware} > \text{Trust-Aware} > \text{UCB} > \text{Explore} > \text{Random}
]

---

## 结果分析

TruthRide 在整个实验过程中表现出最高的累计平台效用。

原因如下：

1. **减少负收益数据**
   Trust机制能够识别低质量数据，从而减少负收益任务；

2. **提升高质量数据比例**
   UCB机制提高高质量工人选择概率；

3. **增强长期供给能力**
   激励机制（PGRD/LGSC）提高可信工人留存率；

4. **形成正反馈循环**
   高质量 → 高收益 → 更多可信工人参与 → 更高质量。
