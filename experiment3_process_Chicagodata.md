# 芝加哥共享单车数据处理流程文档

## 1. 数据来源

原始数据为芝加哥共享单车系统（Divvy）在2020年4月1日的行程记录，文件格式为Excel（`Chicago.xlsx`）。每条记录包含以下字段：

| 字段名 | 类型 | 描述 |
|--------|------|------|
| `ride_id` | 字符串 | 行程唯一标识 |
| `started_at` | 日期时间 | 行程开始时间 |
| `ended_at` | 日期时间 | 行程结束时间 |
| `start_lat`, `start_lng` | 浮点数 | 起点经纬度 |
| `end_lat`, `end_lng` | 浮点数 | 终点经纬度 |

原始数据共包含数千条行程，本研究选取前2000辆唯一车辆（按`ride_id`字典序排序）进行处理。

## 2. 数据处理流程

### 2.1 数据清洗与格式统一

1. **日期筛选**：仅保留`started_at`日期为2020-04-01的记录。
2. **时间转换**：将`started_at`和`ended_at`转换为从午夜0点起的秒数（`start_time`, `end_time`），范围0~86400。
3. **车辆ID重编号**：将原始字符串`ride_id`映射为连续整数1,2,3,…，按字典序排序。
4. **拆分行程为点**：每条行程拆分为起点和终点两行，每行包含`ride_id`, `time`, `lat`, `lng`。输出文件`Chicago_points.csv`。

### 2.2 空间网格划分

为了将连续空间离散化，采用等间距网格将研究区域划分为`10×10`的网格。具体步骤如下：

1. **确定边界**：基于所有点的经纬度，计算最小外接矩形：
   ```
   lon_min = min(lng), lon_max = max(lng)
   lat_min = min(lat), lat_max = max(lat)
   ```
2. **网格步长**：
   ```
   step_lon = (lon_max - lon_min) / 10
   step_lat = (lat_max - lat_min) / 10
   ```
3. **区域ID分配**：对任意点`(lon, lat)`，其网格索引为：
   ```
   gx = floor((lon - lon_min) / step_lon)
   gy = floor((lat - lat_min) / step_lat)
   region_id = gy * 10 + gx
   ```
   边界点被裁剪到最近网格内，确保`region_id`范围0~99。

### 2.3 车辆轨迹段构建

原始行程时间跨度很短（通常几分钟），不利于模拟长时间工人可用性。因此，我们重新为每辆车生成一个**持续6~15小时**的轨迹段，具体规则如下：

- 对每辆车，取其所有点中时间最早的点作为**起始参考点**，其`start_time`即为该车的起始时间（保留原始时间的小时信息），其经纬度用于确定该车所在的网格`region_id`。
- 随机生成持续时间`D`（单位：秒），服从均匀分布`U(6×3600, 15×3600)`。
- 该车的结束时间`end_time = start_time + D`，若超过86400秒（一天结束），则截断为86400。
- 车辆在整个段内被认为**始终停留在起始网格**（即`region_id`不变）。这一假设符合共享单车用户长时间停放车辆的特点，同时也简化了后续任务分配。

### 2.4 时间切片

为支持10分钟一轮的任务招募，将每个车辆段按**600秒（10分钟）** 的固定时间片切分。切片规则如下：

- 对段`[start, end]`，从`start`开始，依次确定每个时间片的起止边界：
  ```
  next_boundary = ((cur // 600) + 1) * 600
  seg_end = min(end, next_boundary - 1)
  ```
- 每个子段继承原段的`region_id`。
- 最终得到每个车辆的一系列10分钟段，每个段表示为`(vehicle_id, region_id, start_time, end_time)`。

### 2.5 工人属性生成

为每辆车生成固定的成本`cost`和可信标识`is_trusted`，用于后续任务分配算法：

- `cost`：均匀分布随机数，范围5.0~20.0，保留1位小数。
- `is_trusted`：伯努利分布，以70%概率为`True`（可信），30%概率为`False`（不可信）。

随机种子固定为42，保证结果可复现。

### 2.6 最终输出文件

所有段按`vehicle_id`和`start_time`排序后保存为CSV文件，列定义如下：

| 列名 | 类型 | 描述 |
|------|------|------|
| `vehicle_id` | int | 车辆唯一编号（1~2000） |
| `region_id` | int | 网格区域编号（0~99） |
| `start_time` | int | 段开始时间（秒，0~86400） |
| `end_time` | int | 段结束时间（秒） |
| `cost` | float | 该工人的报价（5~20） |
| `is_trusted` | bool | 是否可信工人 |

## 3. 数据可视化

为了直观展示数据分布，生成左右双图：

- **左图**：所有点的散点图，叠加10×10网格线和边界框，用于观察空间覆盖。
- **右图**：热力图，每个网格内的点数密度，颜色越深表示该区域被访问越频繁。

输出图片`experiment3_grid_partition.png`。

## 4. 参数配置汇总

| 参数 | 值 | 说明 |
|------|-----|------|
| `GRID_X_NUM`, `GRID_Y_NUM` | 10, 10 | 网格数量 |
| `SLOT_SEC` | 600 | 时间片长度（秒），10分钟 |
| `MIN_DURATION_HOURS` | 6 | 最短轨迹持续时间（小时） |
| `MAX_DURATION_HOURS` | 15 | 最长轨迹持续时间（小时） |
| `COST_MIN`, `COST_MAX` | 5.0, 20.0 | 成本范围 |
| `TRUSTED_RATIO` | 0.7 | 可信工人比例 |
| `MAX_VEHICLES` | 2000 | 最多保留的车辆数 |
| `random.seed` | 42 | 随机种子 |

## 5. 运行与结果

执行脚本`experiment3_process_Chicagodata1.py`后，将生成以下文件：

- `experiment3_vehicle.csv`：车辆段数据，共包含约数万个10分钟段（取决于每辆车覆盖的时间片数）。
- `experiment3_grid_partition.png`：网格分布图。

## 6. 结论

本处理流程将原始共享单车行程数据转换为适合众包任务分配算法的工人轨迹段数据。通过重新分配每辆车的活跃时长（6~15小时），保证了工人在一天中的可用性覆盖，为后续任务生成与匹配提供了合理的时空容量基础。
