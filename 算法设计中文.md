# 第四章 所提多阶段动态优化算法设计

## 4.1 算法总体思路

根据第三章建立的系统模型，本章给出本文提出的求解方法，用于解决群智感知平台中的以下核心问题：

1. 在预算约束下如何高效招募工人；
2. 如何识别高可信工人并抑制恶意工人；
3. 如何提升任务完成率与数据质量；
4. 如何降低优质工人的长期流失率；
5. 如何提高平台长期收益。

由于工人招募、信誉演化、会员激励以及退出行为相互影响，难以通过一次性静态优化直接求解。因此，本文采用**多阶段动态决策机制**，平台在每一轮感知周期中持续更新决策。

每一轮的执行流程如下：

**任务生成 → 会员状态更新 → 工人招募 → 数据上报 → 信誉验证 → 留存更新 → 下一轮**

为便于描述，本文将整体方法划分为四个子算法：

* **算法1：总体动态优化框架**
* **算法2：贪心工人招募算法**
* **算法3：空间一致性信誉更新算法**
* **算法4：会员与留存更新算法**

---

# 4.2 算法1：总体动态优化框架

算法1描述平台在整个运行周期内的总体流程。

系统初始化时，需要设置：

1. 工人初始信誉值；
2. 初始会员状态；
3. 初始沉没参与值；
4. 初始活跃工人集合。

随后平台按轮次持续运行，在每一轮中依次执行任务发布、招募、信誉更新和留存更新。

---

### 算法1 总体动态优化框架

**输入：** 工人集合 $\mathcal{V}$，总轮数 $H$，每轮预算 $B_t$

**输出：** 平台累计收益 $U_{cum}$

```text id="algcn1001"
1:  初始化所有工人信誉值 ri(0)

2:  初始化会员状态 mi(0) ← 0

3:  初始化沉没参与值 Si(0) ← 0

4:  初始化活跃工人集合 Vactive ← V

5:  初始化累计收益 Ucum ← 0

6:  For t = 1 到 H do

7:      生成第 t 轮任务集合 T(t)

8:      执行算法4，更新会员状态

9:      执行算法2，完成工人招募

10:     收集被招募工人的任务数据

11:     执行算法3，更新工人信誉

12:     执行算法4，更新留存与退出状态

13:     计算本轮收益 U(t)

14:     Ucum ← Ucum + U(t)

15: End For

16: Return Ucum
```

其中：

$$
U_{cum}=\sum_{t=1}^{H}U(t)
$$

# 4.3 算法2：贪心工人招募算法

## 4.3.1 设计思想

工人招募是平台最重要的短期决策阶段，直接影响：

1. 本轮完成多少任务；
2. 数据质量高低；
3. 平台支付成本；
4. 后续信誉验证机会。

若只选择低报价工人，则质量可能过低；
若只选择高质量工人，则成本过高。

因此，本文采用**单位成本收益最大化的贪心招募策略**。

---

### 算法2 贪心工人招募算法

**输入：** 候选工人集合 $\mathcal{V}^{cand}(t)$，任务集合 $\mathcal{T}(t)$，预算 $B_t$

**输出：** 招募工人集合 $\mathcal{S}(t)$，支付成本 $P(t)$

```text id="algcn2001"
1:  初始化 S(t) ← ∅

2:  初始化 P(t) ← 0

3:  对每个任务 τj ∈ T(t)：

4:      当前最好质量 Q̂j(t) ← 0

5:  删除所有恶意工人

6:  Repeat

7:      For 每个未被选中的工人 vi do

8:          若 bi > Bt − P(t)，跳过

9:          估计工人质量：

                q̂i(t)=
                { 1, if ni(t)=0
                { min(1, q̄i(t)+sqrt((K+1)ln(max(2,Nobs(t)))/ni(t)))

10:         计算边际收益：

                MGi(t)=
                Στj∈Bi(t)∩T(t)
                wj·max(0,q̂i(t)-Q̂j(t))

11:         计算招募得分：

                Scorei(t)=
                { MGi(t)/bi, if MGi(t)>0
                { 0, otherwise

12:     End For

13:     选择最高得分工人：

                v* = argmax Scorei(t)

14:     若不存在正收益工人，则结束

15:     S(t) ← S(t) ∪ {v*}

16:     P(t) ← P(t) + bv*

17:     更新其覆盖任务的当前质量

18: Until 预算耗尽或无可选工人

19: Return S(t), P(t)
```

该算法优先选择“每单位成本带来最大质量提升”的工人。

# 4.4 算法3：空间一致性信誉更新算法

## 4.4.1 设计思想

平台无法直接知道任务真实值，因此不能直接判断工人数据是否正确。

本文采用：

> **可信工人作为参考锚点，对未知工人进行间接验证。**

若可信工人与未知工人在同一区域执行同类任务，则可比较其结果一致性。

---

### 算法3 空间一致性信誉更新算法

**输入：** 工人上报数据，当前信誉值

**输出：** 更新后的信誉值与工人类别

```text id="algcn3001"
1:  找出同时存在可信工人与未知工人的网格区域

2:  按以下规则排序网格：

        (1) 未知工人数多的优先
        (2) 可信工人数多的优先

3:  选取前 M 个网格作为验证区域

4:  For 每个验证区域 do

5:      选取该区域中的验证任务 τj

6:      收集可信工人上报值

7:      计算参考值：

            ẑj(t)= median(可信工人数据)

8:      For 每个参与该任务的未知工人 vi do

9:          计算误差：

                eij(t)=|yij(t)-ẑj(t)|

10:         更新信誉：

                ri(t+1)=
                { min(1,ri(t)+η), if eij(t)≤εg
                { ri(t), if εg<eij(t)≤εb
                { max(0,ri(t)-η), if eij(t)>εb

11:     End For

12: End For

13: 按信誉重新分类：

        ri ≥ Th     → 可信工人
        ri ≤ Tl     → 恶意工人
        其他        → 未知工人
```

该算法可逐步扩大可信工人集合，并识别恶意工人。

# 4.5 算法4：会员与留存更新算法

## 4.5.1 设计思想

即使工人可信，若长期收益不足，仍可能离开平台。

因此本文引入：

1. 会员机制（提供更多任务机会）；
2. 沉没成本机制（接近奖励时不愿离开）。

---

### 算法4 会员与留存更新算法

**输入：** 活跃工人集合

**输出：** 更新后的活跃工人集合

```text id="algcn4001"
1:  For 每个活跃工人 vi do

2:      若 vi 为可信工人，则判断是否加入会员

3:      若成为会员，则 mi(t)=1

4:      若 mi(t)=1，则更新沉没参与值：

            Si(t+1)=Si(t)+ΔSi(t)

5:      若 Si(t) ≥ Y，则发放奖励 Θ

6:      奖励后重置 Si(t)=0

7:      计算离开概率：

            pleave_i(t)=
            σ(β0+β1Ci(t)-β2R̄i(t)-β3mi(t)-β4Si(t)/Y)

8:      若随机触发离开，则移出平台

9:  End For
```

其中：

* 成本越高，越容易离开；
* 收益越高，越不容易离开；
* 是会员越稳定；
* 越接近奖励阈值越不愿离开。

# 4.6 算法复杂度分析

设：

* $N$ 为工人数；
* $M$ 为每轮任务数；
* $H$ 为总轮数。

则：

### 招募阶段

$$
O(NM)
$$

### 信誉更新阶段

$$
O(N)
$$

### 留存更新阶段

$$
O(N)
$$

因此总体复杂度为：

$$
O(HNM)
$$

能够满足大规模群智感知平台实时运行需求。
