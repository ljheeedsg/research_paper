# 实验一：整体数据质量对比实验报告

## 1. 实验目的

本实验旨在验证 **TruthRide** 在长期群智感知过程中，是否能够比基准方法持续获得更高质量、更低误差的数据。围绕这一目标，实验重点回答以下四个问题：

1. TruthRide 在**每轮平均数据质量**上是否优于基准算法；
2. TruthRide 在**累计平均数据质量**上是否具有长期优势；
3. TruthRide 在**累计 RMSE** 上是否能够更有效地抑制虚假数据带来的整体误差；
4. TruthRide 在**每轮 RMSE** 上的波动是否可解释，且是否仍能保持总体稳定优势。

---

## 2. 实验设置

### 2.1 数据集

实验一基于 **Beijing Taxi / T-Drive** 数据集开展。该数据集轨迹密度较高，能够在相同空间网格中形成较多工人交叉覆盖，因此更适合验证本文所采用的空间一致性信任评估机制。

### 2.2 对比算法

实验比较以下六种方法：

1. **Random**：随机选择可用工人；
2. **Explore-First**：前期随机探索、后期基于历史结果利用；
3. **UCB-Greedy**：基于 UCB 和边际收益的工人选择方法；
4. **Trust-Aware**：在 UCB-Greedy 基础上加入信任验证机制；
5. **Incentive-Aware**：在 Trust 基础上加入激励机制；
6. **TruthRide**：本文完整方法，结合 UCB、Trust、PGRD 与 LGSC。

### 2.3 评价指标

本实验共使用四张图对应四个指标：

1. **Fig. 1(a)**：Per-Round Average Quality；
2. **Fig. 1(b)**：Cumulative Average Quality；
3. **Fig. 1(c)**：Cumulative RMSE；
4. **Fig. 1(d)**：Per-Round RMSE。

其中，Fig. 1(a) 与 Fig. 1(d) 使用滑动平均后的结果，以减弱单轮任务分布差异带来的剧烈波动，更清楚地展示长期趋势。

---

## 3. 实验结果与分析

## 3.1 Fig. 1(a)：每轮平均数据质量分析

Fig. 1(a) 展示了不同算法在各轮中的平均数据质量变化趋势。从整体曲线可以看出，TruthRide 在大多数轮次中始终处于最高位置，并且中后期表现尤为稳定。

从具体趋势看，Random 的质量始终处于较低水平，基本维持在 `0.52-0.60` 区间，说明随机招募无法有效区分高质量工人与低质量工人。Explore-First 在前中期一度提升较快，但后期快速下降，甚至在后半段接近于失效，反映出“先探索后利用”策略一旦前期学习结果受到噪声或恶意数据干扰，后期就可能持续锁定次优工人。UCB-Greedy 相比 Random 明显更优，说明基于历史反馈进行招募优化能够提升数据质量，但由于缺乏显式的虚假数据识别机制，其曲线在中后期仍存在明显回落。Trust-Aware 加入信任验证后，整体水平显著高于 UCB-Greedy，说明利用可信参考值筛除高偏差工人能够有效改善平台收到的数据质量。Incentive-Aware 在此基础上进一步提升，表明激励机制有助于提高高质量工人的参与概率。TruthRide 则在多数轮次上持续优于其他算法，后期始终稳定在 `0.77-0.82` 左右，显示出完整机制在长期运行中的稳定性最强。

这说明，单靠“选择历史高质量工人”并不足以保证长期数据质量；只有同时结合信任验证、参与激励与留存机制，平台才能持续获得更高质量的数据来源。

## 3.2 Fig. 1(b)：累计平均数据质量分析

Fig. 1(b) 反映了从第 1 轮累积到当前轮次时，平台整体获得的数据质量水平。与每轮指标相比，累计平均数据质量更能体现算法的长期表现。

从结果看，TruthRide 的累计平均数据质量始终最高，并且领先优势在中后期保持稳定。实验结束时，TruthRide 的最终累计平均数据质量达到 **0.7950**，明显高于 Incentive-Aware 的 **0.7543**、Trust-Aware 的 **0.7292**、UCB-Greedy 的 **0.5847**、Explore-First 的 **0.5631** 和 Random 的 **0.5540**。相比最强基线 Incentive-Aware，TruthRide 进一步提升约 **0.0407**；相比 Trust-Aware 提升约 **0.0658**；相比 UCB-Greedy 提升约 **0.2103**。

这一结果说明，TruthRide 的优势并不是来自少数轮次的偶然高值，而是在长期运行中不断积累形成的稳定优势。其原因在于：UCB 负责从候选工人中优先选择历史表现更好的工人；Trust 机制持续抑制与可信参考值偏差较大的工人；PGRD 与 LGSC 则进一步提升高质量工人的参与意愿和留存能力。三者共同作用，使得平台在长期过程中逐步形成更稳定的高质量工人池，因此累计数据质量能够持续领先。

## 3.3 Fig. 1(c)：累计 RMSE 分析

Fig. 1(c) 从误差角度验证了本文方法的有效性。RMSE 越低，说明工人上报数据越接近可信参考值，平台最终获得的数据越可靠。

从曲线看，TruthRide 的累计 RMSE 全程最低，且随轮次推进逐步下降并保持稳定。实验结束时，TruthRide 的最终累计 RMSE 为 **0.0745**，显著低于 Trust-Aware 的 **0.1403**、UCB-Greedy 的 **0.2437** 和 Random 的 **0.2525**。与 Trust-Aware 相比，TruthRide 将累计 RMSE 进一步降低约 **46.9%**；与 UCB-Greedy 相比降低约 **69.4%**；与 Random 相比降低约 **70.5%**。

这一结果表明，TruthRide 不仅能够提高数据质量评分，更重要的是能够显著降低真实上报值与可信参考值之间的偏差。UCB-Greedy 虽然具备一定学习能力，但无法稳定识别伪造或高偏差数据，因此累计误差仍然较高。Trust-Aware 通过空间一致性验证识别并抑制异常数据，使 RMSE 明显下降。TruthRide 在此基础上通过激励和留存机制持续保留可信工人，使高质量数据来源更加稳定，因此最终获得最低的长期累计误差。

## 3.4 Fig. 1(d)：每轮 RMSE 分析

Fig. 1(d) 展示了不同算法在每一轮中的即时误差表现。与累计 RMSE 相比，该指标对单轮任务组成和异常样本更加敏感，因此曲线存在局部波动是合理现象。

整体来看，TruthRide 的每轮 RMSE 在大多数轮次中均保持最低或接近最低水平，说明本文方法在单轮任务中通常也能获得更接近可信参考值的数据。尤其是在前中期，TruthRide 的 RMSE 下降速度明显快于其他方法，表明系统能够较早建立起更可靠的工人选择与验证机制。

需要说明的是，在约第 `110` 轮附近，TruthRide 曲线出现了一次局部上升。结合原始数据检查可知，这一现象并不意味着算法整体失效，而主要是由单轮异常样本引起。由于 Fig. 1(d) 采用了 `5` 轮滑动平均，该局部抬升实际上主要受到第 `113` 轮异常误差的影响。在该轮中，一名新进入且尚未充分验证的工人执行了任务 `t35_013`，其上报值与可信参考值之间存在较大偏差，从而显著抬高了当轮 RMSE。由于 Per-Round RMSE 本身对单轮异常值高度敏感，因此这种短时波动是可以预期的。

但更重要的是，这种局部波动并未改变 TruthRide 的整体优势。一方面，该上升仅出现在个别轮次；另一方面，Fig. 1(c) 的累计 RMSE 仍然持续下降并始终保持全局最优。这说明 TruthRide 即使在个别轮次受到异常工人干扰，也能够依靠后续的信任更新与长期筛选机制恢复稳定，从整体上维持最低误差水平。

---

## 4. 实验结论

通过 Fig. 1(a)-Fig. 1(d) 四张图，可以得到以下结论：

1. **TruthRide 在每轮平均数据质量上整体最优。** 相比 Random、Explore-First 和 UCB-Greedy，TruthRide 能在大多数轮次中持续获得更高质量的数据；相比 Trust-Aware 和 Incentive-Aware，TruthRide 的中后期表现更稳定。

2. **TruthRide 在长期累计数据质量上具有最强优势。** 最终累计平均数据质量达到 **0.7950**，显著高于所有对比方法，说明本文方法不是短期偶然占优，而是能够在长期运行中稳定积累高质量数据。

3. **TruthRide 在误差控制上显著优于基准方法。** 最终累计 RMSE 仅为 **0.0745**，明显低于 Trust-Aware、UCB-Greedy 和 Random，证明本文方法能够更有效地抑制虚假数据和高偏差数据的影响。

4. **局部 RMSE 波动是可解释的瞬时现象，不影响整体结论。** 第 `110` 轮附近的短时上升主要由个别尚未充分验证的新工人带来的异常误差引起，而不是算法后期整体退化。结合累计 RMSE 曲线可知，TruthRide 仍保持了最优的长期误差控制能力。

综上，实验一充分证明了：**TruthRide 通过“高质量工人选择 + 信任验证 + 激励参与 + 长期留存”的联合机制，能够在长期群智感知过程中持续获得更高质量、更低误差且更稳定的数据，是本文在数据质量维度上的最优方案。**

**这张图说明什么：**

Fig. 1(d) 展示了不同算法在每一轮中的即时误差变化情况，用于观察算法在动态环境下对单轮异常数据的敏感性与鲁棒性。
