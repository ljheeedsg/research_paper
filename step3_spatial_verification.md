 # 总体思路
 我们在发布任务后，就按工人报的信息去选取工人，做一个工人的选取算法，要选取的工人尽可能少，成本低，任务被完成率高。
 然后，我们再发布第二次任务，让被选中的工人再选取一次任务。这次的任务即可能是新增加的任务，但最主要的是我们新产生的我们称为验证任务。验证任务是起什么作用的呢？验证任务是我们收到选中工人的轨迹后，我们把网络分成大小相等的grid。我们一个一个grid来计算，这些grid中有多少个工人经过。在开始的时候，我们知道一小部分工人是可信的, 其集合记为Uc，其它的工人的信任度是未知的，不确定其信任度，其集合记为Uu。然后，我们看这些可信工人所经过的grid，Uu中人工人也经过这个grid最多的top 比如ℳ个grid选取出来。这些grid就是有可信工人经过，而且有信任度未定工人经过此grid的人数最多的前ℳ个grid。然后，我们将这ℳ个grid也要采集数据，这些grid的数据采集任务就称为验证任务。验证任务并不是真正的任务，只是为了验证工人的信任度的。可信工人采集了这个grid的数据，而且信任度未确定工人也采集了这个grid的数据。则可以通过对比数据。如果信任度未定工人的数据与可信工人的数据相差非常小，小于一个阀值 ，则提高信任度未确定工人的信任度，反之降低信任度未确定工人的信任度。
平台，将验证任务ℳ个，以及新产生的真任务都发布出去，只让选中的工人选取。这样的任务容易完成，是在工人顺路的路上顺带完成的，因而成本低。
选中的工人再次选取任务后，去做任务。
工人上报数据，
平台更新工人的信任度。如果信任度未确定工人的数据与可信工人一致，则提高其信任度，如果其信任度超过预定的阀值，则归入到可信工人集合Uc。又可以检验别人工人。如果其信任度下降到低于低的阀值后，则归入不可信集合, 或者称为恶意工人集合Um，其集合中的工人不会被选取的。

经过上述过程后，就能够选取到可信工人了，提高数据质量了。

### 空间一致性验证

**问题**：有些工人是"假的"（乱报数据），怎么识别他们？

**答案**：通过空间一致性验证

```python
# spatial_verification.py

def spatial_verification(workers, grid_size=10):
    """
    空间一致性验证算法
    
    原理：
    - 将地图划分为 N×N 的网格
    - 可信工人(Uc)的行为模式作为"标准答案"
    - 验证未验证工人(Uu)的行为是否与标准一致
    
    输入：
        workers: 所有工人的轨迹数据
        grid_size: 网格大小（如10表示10×10网格）
    
    输出：
        trusted_workers: 更新后的可信工人集合
        suspicious_workers: 可疑工人列表
    """
    
    # 第一步：划分网格
    #   地图1500×1000，划分为10×10网格
    #   每个格子：150×100
    
    # 第二步：统计可信工人的访问分布
    #   建立"可信工人密度图"
    #   记录每个格子被多少可信工人访问过
    
    # 第三步：计算验证分数
    #   对于每个未验证工人：
    #   - 统计他访问的格子
    #   - 对比可信工人的密度分布
    #   - 重合度高 → 可信；重合度低 → 可疑
    
    # 第四步：更新信任度
    #   分数 > 阈值 → 加入可信集合
    #   分数 < 阈值 → 标记为可疑
    
    return trusted_workers, suspicious_workers
