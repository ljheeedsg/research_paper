# 第二步：产生任务数据（完整代码对应版文档）

---

# 1. 实验目标

本步骤的目标是：基于第一步生成的工人轨迹段数据，按照工人在不同空间区域和时间片中的可用容量，自动生成群智感知任务数据集。

生成的任务应满足以下原则：

* 任务只出现在有工人可执行的位置；
* 工人多的区域与时间段，更容易生成任务；
* 某位置任务数量不超过该位置工人容量；
* 每个任务具有时间、空间、需求人数、价值权重等属性。

本步骤输出的数据将用于后续：

* 工人招募算法（CMAB / 贪心 / 对比算法）
* 工人选择与预算分配
* 任务完成率统计
* 激励机制实验

---

# 2. 输入数据

输入文件来自第一步输出：

```text
experiment2_vehicle.csv
```

该文件记录工人在各时间段、各区域中的轨迹段信息。

至少需要以下字段：

| 字段名        | 含义     |
| ---------- | ------ |
| vehicle_id | 工人编号   |
| region_id  | 区域编号   |
| start_time | 可用开始时间 |

若缺少上述字段，程序报错终止。

# 6. 详细实现逻辑

---

# 6.1 读取工人轨迹文件

读取：

```text
experiment2_vehicle.csv
```

逐行提取：

* `vehicle_id`
* `region_id`
* `start_time`

并将时间转换为所属时间片。

---

# 6.2 时间片编号计算

函数：

```python
slot_id = t_seconds // SLOT_SEC
```

即：

$slot_id=\left\lfloor \frac{time}{600} \right\rfloor$

例如：

| 时间（秒） | slot_id |
| ----- | ------- |
| 0     | 0       |
| 599   | 0       |
| 600   | 1       |
| 1200  | 2       |

---

# 6.3 统计时空工人容量

程序统计每个：

```text
(region_id, slot_id)
```

位置中出现过多少不同工人。

即：

[
capacity(region,slot)=\text{unique workers}
]

代码内部使用集合去重：

```python
capacity[region][slot] = {vehicle_id,...}
```

最终转化为数量：

```python
capacity_count[region][slot]
```

---

# 6.4 统计区域活跃密度

同时统计：

```python
region_density[region_id] += 1
```

表示该区域轨迹段记录数量，用于后续画图。

注意：

> 这不是唯一工人数，而是轨迹段活跃度。

---

# 6.5 构建候选任务位置

所有满足：

$capacity(region,slot)>0$

的位置都可生成任务。

形成候选集合：

```text
(region_id, slot_id, capacity)
```

例如：

```text
(25, 18, 4)
```

表示：

* 区域 25
* 第18个时间片
* 有4名工人可执行任务

---

# 7. 任务生成机制（核心）

---

# 7.1 按容量加权抽样

程序从候选位置中随机抽取任务位置：

```python
random.choices(candidate_slots, weights=capacity)
```

即：

> 容量越大的位置，被选中的概率越高。

因此：

* 工人多的区域更容易生成任务；
* 热点区域任务更多。

---

# 7.2 任务数量约束

若某位置容量为：

$cap$

则该位置生成任务数最多为：

[
cap
]

代码限制：

```python
generated_count[(region,slot)] >= cap
```

则不再生成。

这保证：

> 同一时空位置任务数不会超过工人供给能力。

---

# 7.3 工人数需求

代码：

```python
max_req = min(MAX_REQUIRED_WORKERS, cap)
required_workers = randint(1,max_req)
```

由于：

```python
MAX_REQUIRED_WORKERS = 1
```

因此当前实验所有任务均为：

$required_workers = 1$

---

# 7.4 任务价值权重

先随机生成基础权重：

$base_weight \sim U(1.0,3.0)$

再加入需求人数奖励：

$weight = base + 0.2(required_workers-1)$

最后限制最大值为：

$3.0$

由于当前 `required_workers=1`，因此：

$weight = base_weight$

即当前实验中任务权重就是：

$U(1.0,3.0)$

---

# 7.5 任务时间区间

若任务属于：

```text
slot_id
```

则：

开始时间：

$start = slot_id \times 600$

结束时间：

$end = (slot_id+1)\times 600 -1$

例如：

| slot_id | 时间段         |
| ------- | ----------- |
| 0       | 0 ~ 599     |
| 1       | 600 ~ 1199  |
| 2       | 1200 ~ 1799 |

采用闭区间整数时间表示。

---

# 7.6 任务编号生成

代码格式：

```python
task_id = f"t{region_id:02d}_{seq:03d}"
```

例如：

```text
t03_000
t03_001
t25_007
t99_023
```

含义：

* 前半部分表示区域编号
* 后半部分表示该区域任务序号
