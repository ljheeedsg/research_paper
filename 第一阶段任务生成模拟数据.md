# 第一阶段：生成模拟数据

> 目标：用代码生成模拟数据，验证整个数据流程能跑通

---

## 一、需要生成的数据文件

| 数据文件 | 说明 | 格式 | 产出脚本 |
|----------|------|------|----------|
| `vehicles.csv` | 900辆车的原始轨迹 | CSV | `generate_vehicles.py` |
| `tasks.csv` | 120个任务 | CSV | `generate_tasks.py` |
| `worker_segments.json` | 车辆轨迹时空段（按区域分组） | JSON | `extract_segments.py` |
| `task_segments.json` | 任务时空段（按区域分组） | JSON | `build_task_segments.py` |

---

## 二、vehicles.csv（车辆原始轨迹）

### 字段定义

```
vehicle_id,region_id,start_time,end_time,cost,is_trusted
v00_000,0,120,380,12.5,True
v00_000,2,420,680,12.5,True
v00_001,1,300,520,8.3,False
v00_001,3,560,800,8.3,False
...
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `vehicle_id` | string | 车辆ID，格式：`v{hour}_{idx}`，如 `v00_000` |
| `region_id` | int | 访问的区域ID，0~5 |
| `start_time` | int | 进入区域的时间（秒） |
| `end_time` | int | 离开区域的时间（秒） |
| `cost` | float | 车辆报价，5~20 |
| `is_trusted` | bool | 是否为可信工人（真/假），70%True / 30%False |

### 生成规则

```
- 每辆车随机访问 2~4 个区域
- 进入区域时间在整点后 0~600秒 内随机
- 在每个区域停留 60~300秒
- 车辆报价在 [5, 20] 区间随机
- 车辆可信度：70%为真、30%为假
```

### 统计要求

- 总记录数 ≈ 2000~3000 条（每辆车访问2~4个区域）
- 独立车辆数 = 900
- 真工人比例 ≈ 70%

---

## 三、tasks.csv（任务集合）

### 字段定义

```
task_id,region_id,start_time,end_time,required_workers
t00_00,0,0,3600,2
t00_01,0,500,4100,1
t01_00,1,3600,7200,2
...
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `task_id` | string | 任务ID，格式：`t{region}_{idx}`，如 `t00_00` |
| `region_id` | int | 任务所在区域，0~5 |
| `start_time` | int | 任务开始时间（秒） |
| `end_time` | int | 任务结束时间（秒） |
| `required_workers` | int | 完成任务需要的工人数，1~3 |

### 生成规则

```
- 每个区域分配 20 个任务（6区域 × 20 = 120）
- 任务时间窗口 = 3600秒（1小时）
- 任务随机分配到6小时中的任意时段
- 每个任务需要 1~3 个工人完成
```

### 统计要求

- 总记录数 = 120
- 每个区域任务数 = 20
- 时间窗口 = 3600秒（1小时）
- 覆盖整个6小时（0~21600秒）
