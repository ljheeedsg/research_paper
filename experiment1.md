# 9.2 消融实验设计（Ablation Study）
## 9.2.1 实验目的
本实验采用**梯度式模块叠加消融**，在完全统一的环境下，依次引入 **CMAB 贪心招募、动态信任验证、PGRD会员激励、LGSC长期留存激励**，量化验证每个模块对系统性能的贡献。
重点验证：
1. CMAB 招募是否显著优于随机招募；
2. **动态信任机制**能否从**未知工人 Uu**中正确识别**可信 Uc**与**恶意 Um**；
3. 信任机制排除恶意工人后，**任务覆盖率为何先小幅下降、后期大幅反超**；
4. PGRD 能否提升可信工人入会率与平台效用；
5. LGSC 能否提升会员长期留存；
6. 完整方案是否实现**信任筛选—招募优化—短期质量—长期覆盖**的闭环。

# 9.2.2 实验环境与统一控制变量
所有对比组**完全相同**，保证公平可复现：
- **数据集**：罗马出租车轨迹，156 名工人，200 个时空任务
- **初始工人分类**：仅 **可信 Uc + 未知 Uu**，无预先恶意工人
- **攻击设置**：20% 未知工人为隐藏攻击者（10% 持续攻击、10% 间歇攻击）

- **实验轮次**：24 轮（1 轮/小时）
- **重复次数**：100 次取平均，剔除异常值
- **随机种子**：42
- **硬件环境**：Intel i7，16GB RAM
- **软件环境**：Python 3.9，NumPy 1.24，Matplotlib 3.7
- **所有参数**：使用论文默认最优参数

# 9.2.5 核心评价指标（完整公式+定义）
## （1）招募性能
1. **任务覆盖率**
$$Coverage = \frac{covered\_task\_count}{200}$$
2. **单位预算覆盖效率**
$$CostEffi = \frac{covered\_task\_count}{total\_cost}$$

## （2）动态信任质量
3. **恶意识别数量**：最终从 Uu 中识别出的 Um 数量
4. **可信扩增比例**
$$TrustedGrowth = \frac{|Uc_{final}|}{|Uc_{init}|}$$
5. **数据误差 RMSE**
$$RMSE = \sqrt{\frac{1}{N}\sum (data_{i,j}-truth_j)^2}$$

## （3）激励与平台效用
6. **会员入会率**
$$JoinRate = \frac{member\_count}{|Uc|}$$
7. **会员留存率**（连续≥10轮）
$$Retention = \frac{members_{≥10}}{total\_members}$$
8. **平台总效用**
$$Util = 系统收益 + 会费 - 总成本 - 奖励金$$
9. **高质量数据率**（质量≥0.8）
$$HighQuality = \frac{tasks(q≥0.8)}{completed\_tasks}$$
