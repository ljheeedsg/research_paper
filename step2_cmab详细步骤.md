2. **计算工人可选项的总成本**：
   - 总成本 = 该工人能覆盖的所有任务的报价之和
   - 覆盖条件：工人与任务「区域相同 + 时间有交集」
   - 每个任务对该工人有单独报价，累加得到招募该工人的总花费
3. **计算任务完成质量**：
   - 对每个任务 j：$q_{i,j} = \frac{\text{工人i与任务j的时间交集时长}}{3600}$
   - 区域不匹配/时间无重叠 → q=0
   - 结果范围：0 ≤ q ≤ 1
4. **计算任务权重**：
   - 任务 j 的权重 $w_j$ = 任务原始数据中的`required_workers`值

# 第二步：初始化 + 贪心迭代招募 step2_final_recruit.py

## 输入
- `step2_worker_option_set.json`
- `step2_task_weight_list.json`

## 核心流程

### 1. 初始化阶段（t=0，探索阶段）

- **预设参数**：每轮招募人数 `K`，总预算 `B`（可调）。 
- **招募所有工人**：支付每个工人的总成本（覆盖任务报价之和），记录总花费和剩余预算。
- **计算工人档案**：
  - 平均感知质量：`avg_quality = 总质量 / 可覆盖任务数`，其中总质量为该工人所有可覆盖任务的 `quality` 之和。
  - 学习次数 `n_i`：初始化为该工人可覆盖的任务数（因为首次招募后，其所有覆盖任务的质量都会被学习）。
  - 判断次数（用于统计）：初始化为 1。
- **注意**：此阶段**不更新任务覆盖状态**，仅用于学习工人质量，避免过早消耗任务覆盖。

### 2. 贪心选择阶段（利用阶段）

- **每轮选择**：从所有工人中（可重复招募）选择 `K` 个工人，每个工人仅在该轮内最多选一次。
- **选择标准（论文公式 12）**：  
  对每个候选工人 `i`，计算其边际质量增益与成本的比值：

  其中：
- `UCB质量增益` = 该工人覆盖的**未完成任务**的总加权质量，任务质量用工人的 **UCB 质量估计**代替真实质量。
- `cost_i` = 工人的总成本（`total_cost`）。

- **UCB 质量估计（论文公式 10）**：

- `avg_quality_i(t-1)`：截至上一轮的平均感知质量。
- `n_i(t-1)`：工人 `i` 被学习的次数（所有被选中选项覆盖的任务数之和）。
- `Σ n_i'(t-1)`：所有工人学习总次数。

- **工人档案更新（论文公式 8、9）**：
- 当工人 `i` 被选中，其学习次数 `n_i` 增加 `|M_i^l|`（该工人覆盖的任务数）。
- 平均质量更新：根据新观测到的质量（模拟为工人当前平均质量）进行加权平均。
- 总学习次数更新。

- **任务覆盖更新**：每轮选中的工人，其覆盖的所有任务标记为“已覆盖”，后续轮次不再计入增益。

- **终止条件**：
- 剩余预算 < 所有工人中的最小成本。
- 所有工人的边际增益为 0（即无未覆盖任务）。
- （可选）达到指定轮数上限（本例未设置，由预算自然终止）。
